## Project Overview

ArcheryCounter is a C++17 computer vision application that detects arrows in archery target photos and scores them automatically. It uses OpenCV for image processing.

## Build & Run

OpenCV must be installed. On macOS: `brew install opencv`. On Linux: `sudo apt-get install libopencv-dev`.

```bash
mkdir build && cd build
cmake ..
cmake --build . -- -j$(nproc)
./ArcheryCounter
```

On Linux CI, the build uses `scan-build` (clang static analyzer) wrapping the build step:
```bash
scan-build cmake --build . -- -j2
```

The `sources.hpp` file is auto-generated by CMake from `sources.hpp.in` ‚Äî it defines `IMAGE_TEST_DIR` pointing to the `Images/` directory. Do not edit `sources.hpp` directly.

There are no automated tests; the binary runs on hardcoded test images and displays results via OpenCV windows.

## Architecture

The pipeline in `main.cpp` has two stages:

**Stage 1 ‚Äî Target detection** (`ellipses` namespace in `ellispes_approach.*`):
- `ellipses::find_target()` returns `std::array<cv::RotatedRect, 10>` ‚Äî the 10 concentric ring ellipses of the target
- Internally: applies `filter_image()` (5 HSV colour filters: white/yellow/red/blue/black) to a preprocessed image, then uses `std::async` to detect ellipses from the first 3 colour-filtered images in parallel via `detect_circle_approach()`
- The 3 candidate ellipses are scored by `find_weakest_element()` (compares center, angle, aspect ratio) and the worst is discarded; `compute_final_ellipses_by_linear_interpolation()` then interpolates all 10 rings from the 2-3 good ones
- `compute_target_ring_mask()` in `main.cpp` converts these ellipses into filled binary masks

**Stage 2 ‚Äî Arrow detection** (`NN` namespace in `NN_approach.hpp`):
- `NN::find_arrows()` returns `arrows_position` (tip and fletching `cv::Point` vectors) ‚Äî currently a stub returning empty results
- Arrow scoring: `get_arrow_point()` checks each tip against the 10 ring masks walking outermost-to-innermost; returns 0 for a miss (outside all rings), up to 10 for a bullseye

**Supporting modules:**
- `pretreatment.*` ‚Äî image preprocessing (grayscale, blur, etc.) before detection
- `colour_filters.*` ‚Äî HSV-space filters for white/yellow/red/blue/black target ring colors; `filter_arrow()` isolates arrow pixels; `filter_image()` returns all 5 as `std::array<cv::Mat, 5>`
- `hough_approach.*`, `histo_approach.*`, `keypoint_approach.*` ‚Äî alternative detection approaches (not used in main pipeline)
- `math_utils.hpp` ‚Äî template utilities: `euclideanDistance`, `center_of_mass`, `mean_distance`

## Key Design Notes

- The 10-ring mask array is ordered so that mask index 0 = outermost ring; scoring subtracts from 10 as it walks inward
- `NN_approach.hpp` is entirely header-only and its `find_arrows` implementation is a placeholder stub
- The hardcoded `model` and `test` image paths in `main.cpp:11-12` use raw string literals with Windows-style backslash separators ‚Äî these need adjustment for cross-platform use
- `Images/model/cleaned.jpg` is a clean reference target image; `Images/` contains 10 test photos
