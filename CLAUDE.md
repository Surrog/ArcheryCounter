## Project Overview

ArcheryCounter is a C++17 computer vision application that detects arrows in archery target photos and scores them automatically. It uses OpenCV for image processing.

## Build

OpenCV must be installed. On macOS: `brew install opencv`. On Linux: `sudo apt-get install libopencv-dev`.

```bash
mkdir build && cd build
cmake ..
cmake --build . -- -j$(nproc)
```

On Linux CI, the build uses `scan-build` (clang static analyzer) wrapping the build step:
```bash
scan-build cmake --build . -- -j2
```

The `sources.hpp` file is auto-generated by CMake from `sources.hpp.in` — it defines `IMAGE_TEST_DIR` pointing to the `Images/` directory. Do not edit `sources.hpp` directly.

## Code Style

Formatting is enforced via `_clang-format` (WebKit-based style):
- 4-space indentation, no tabs
- 120 column limit
- Allman brace style
- No space before parentheses

Format with: `clang-format -i *.cpp *.hpp`

## Architecture

The pipeline in `main.cpp` has two stages:

**Stage 1 — Target detection** (`ellipses` namespace in `ellispes_approach.*`):
- `ellipses::find_target()` returns `std::array<cv::RotatedRect, 10>` — the 10 concentric ring ellipses of the target
- Uses contour detection, center estimation, and linear interpolation to produce all 10 rings from a few detected ones
- `compute_target_ring_mask()` in `main.cpp` converts these ellipses into filled binary masks

**Stage 2 — Arrow detection** (`NN` namespace in `NN_approach.hpp`):
- `NN::find_arrows()` returns `arrows_position` (tip and fletching `cv::Point` vectors) — currently a stub returning empty results
- Arrow scoring: each arrow tip is checked against the 10 ring masks (outermost-first order) via `get_arrow_point()`

**Supporting modules:**
- `pretreatment.*` — image preprocessing (grayscale, blur, etc.) before detection
- `colour_filters.*` — HSV-space filters for white/yellow/red/blue/black target ring colors; `filter_arrow()` isolates arrow pixels
- `hough_approach.*`, `histo_approach.*`, `keypoint_approach.*` — alternative detection approaches (not used in main pipeline)
- `math_utils.hpp` — template utilities: `euclideanDistance`, `center_of_mass`, `mean_distance`

## Key Design Notes

- The 10-ring mask array is ordered so that mask index 0 = outermost ring; scoring subtracts from 10 as it walks inward
- `NN_approach.hpp` is entirely header-only and its `find_arrows` implementation is a placeholder stub
- The hardcoded `model` and `test` image paths in `main.cpp` use raw string literals with Windows-style backslash separators — these need adjustment for cross-platform use
