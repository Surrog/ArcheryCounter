# JavaScript / React Native Migration Guide

This document describes how the C++ OpenCV archery scoring code is integrated into a React Native app.

## Architecture

The C++ algorithm files are compiled into a **platform native module** (iOS and Android) that the React Native JS layer calls via a `Promise`-based bridge. No JavaScript reimplementation of the OpenCV algorithms is needed.

```
┌─────────────────────────────────────────────┐
│           React Native (JS/TS)              │
│  HomeScreen → useArcheryScorer              │
│  → NativeModules.ArcheryCounter             │
└───────────────┬─────────────────────────────┘
                │  Promise<RingEllipse[]>
    ┌───────────┴────────────┐
    │  iOS (.mm)             │  Android (JNI / Java)
    │  RCTArcheryCounter     │  ArcheryCounterModule
    └───────────┬────────────┘
                │
    ┌───────────┴────────────────────────────────┐
    │  cpp/ArcheryCounterModule.cpp              │
    │  processImageFile(path) → ArcheryResult    │
    └───────────┬────────────────────────────────┘
                │
    ┌───────────┴────────────────────────────────┐
    │  Existing C++ algorithm files (repo root)  │
    │  ellipses::find_target()                   │
    │  colour_filters, pretreatment, math_utils  │
    └────────────────────────────────────────────┘
```

## File Map

| File | Purpose |
|------|---------|
| `cpp/ArcheryCounterModule.hpp` | Platform-independent C++ API (`processImageFile`) |
| `cpp/ArcheryCounterModule.cpp` | Calls `ellipses::find_target`, returns `ArcheryResult` |
| `ios/ArcheryCounterBridge/RCTArcheryCounter.{h,mm}` | iOS Objective-C++ native module |

## One-Time Setup

### 1. Initialize the native project

The `ios/` and `android/` directories are generated by the React Native CLI. Run this
from the repo root (the existing `package.json` is used as-is):

```bash
npx react-native@0.84.1 init ArcheryCounterApp --skip-install
# Then move the generated ios/ and android/ dirs into the repo root,
# overwriting the template App.tsx with ours.
```

### 2. Install JS dependencies

```bash
npm install
```

### 3. iOS — OpenCV via CocoaPods

The podspec (`ArcheryCounterNative.podspec`) declares `pod 'OpenCV2', '~> 4.8'`.
Add the local pod to `ios/Podfile`:

```ruby
platform :ios, '13.0'

target 'ArcheryCounterApp' do
  config = use_native_modules!
  use_react_native!(...)

  pod 'ArcheryCounterNative', :path => '../'
end
```

Then:

```bash
cd ios && pod install
```

**Required Xcode project settings** (via the podspec's `pod_target_xcconfig`):
- `HEADER_SEARCH_PATHS` includes the repo root and `cpp/`
- `CLANG_CXX_LANGUAGE_STANDARD = c++17`

### 4. Android — OpenCV SDK

Download the OpenCV Android SDK from https://opencv.org/releases/ (4.x release,
e.g. `OpenCV-4.9.0-android-sdk.zip`) and unzip it to:

```
/Users/francoisancel/Documents/OpenCV-android-sdk/
```

Or set the path via a Gradle argument in `android/app/build.gradle`:

```gradle
android {
    defaultConfig {
        externalNativeBuild {
            cmake {
                arguments "-DANDROID_STL=c++_shared",
                          "-DOPENCV_ANDROID_SDK=/your/path/to/OpenCV-android-sdk"
                abiFilters "arm64-v8a", "armeabi-v7a"
            }
        }
    }
    externalNativeBuild {
        cmake {
            path "src/main/jni/CMakeLists.txt"
            version "3.22.1"
        }
    }
    sourceSets {
        main {
            // Package libopencv_java4.so into the APK
            jniLibs.srcDirs = ['/your/path/to/OpenCV-android-sdk/sdk/native/libs']
        }
    }
}
```

**Why `ANDROID_STL=c++_shared`:** Both `ellispes_approach.cpp` and
`colour_filters.cpp` use `std::async`. The OpenCV `.so` also links libc++.
Using `c++_shared` prevents ODR (one-definition-rule) violations between
the two translation units. `libc++_shared.so` is packaged automatically.

Register the package in `android/app/src/main/java/com/helloworld/MainApplication.java`:

```java
@Override
protected List<ReactPackage> getPackages() {
    List<ReactPackage> packages = new PackageList(this).getPackages();
    packages.add(new ArcheryCounterPackage());
    return packages;
}
```

### 5. iOS camera/photo library permissions

Add to `ios/ArcheryCounterApp/Info.plist`:

```xml
<key>NSPhotoLibraryUsageDescription</key>
<string>Select a photo of your archery target to score it</string>
<key>NSCameraUsageDescription</key>
<string>Take a photo of your archery target to score it</string>
```

### 6. Android photo permissions

Add to `android/app/src/main/AndroidManifest.xml`:

```xml
<uses-permission android:name="android.permission.READ_MEDIA_IMAGES" />
<!-- For Android < 13: -->
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" android:maxSdkVersion="32" />
```

## Native Module Interface

The module is accessed as `NativeModules.ArcheryCounter` and exposes one method:

```typescript
processImage(imageUri: string): Promise<RingEllipse[]>
```

`imageUri` — a `file://` URI from `react-native-image-picker` (both platforms).
On Android, `content://` URIs are resolved to a temp file automatically by the Java wrapper.

`RingEllipse[]` — array of 10 elements, index 0 = outermost ring, index 9 = bullseye:

```typescript
interface RingEllipse {
  centerX: number;  // original image pixels
  centerY: number;
  width: number;    // full bounding box (semi-axis rx = width  / 2)
  height: number;   // full bounding box (semi-axis ry = height / 2)
  angle: number;    // degrees — same as cv::RotatedRect::angle
}
```

## SVG Overlay and Coordinate Scaling

The overlay (`RingOverlay.tsx`) must account for the letterboxing introduced by
`<Image resizeMode="contain">`. The transform applied before drawing each ellipse:

```
imageAspect  = imageNaturalWidth / imageNaturalHeight
viewAspect   = viewWidth / viewHeight

if imageAspect > viewAspect:          # letterbox top/bottom
    renderedWidth  = viewWidth
    renderedHeight = viewWidth / imageAspect
    offsetX = 0
    offsetY = (viewHeight - renderedHeight) / 2
else:                                 # pillarbox left/right
    renderedHeight = viewHeight
    renderedWidth  = viewHeight * imageAspect
    offsetX = (viewWidth - renderedWidth) / 2
    offsetY = 0

scale = renderedWidth / imageNaturalWidth

svgCenterX = ring.centerX * scale + offsetX
svgCenterY = ring.centerY * scale + offsetY
svgRx      = (ring.width  / 2) * scale
svgRy      = (ring.height / 2) * scale
```

The `<G rotation={ring.angle} origin={cx,cy}>` wrapper on each SVG `<Ellipse>` applies
the rotation around the ellipse center, matching `cv::RotatedRect`'s angle convention.

## Arrow Detection (Future Work)

`NN::find_arrows()` is a stub that returns empty results. When implemented, the native
module can be extended to also return:

```typescript
interface ArrowResult {
  tip: { x: number; y: number };
  score: number;  // 0 = miss, 1–10 = ring score (10 = bullseye)
}
```

The scoring logic (`get_arrow_point` in `main.cpp`) is already implemented in C++ and
can be wired into `cpp/ArcheryCounterModule.cpp` once arrow detection is functional.
